5.2 Detection Module B
Unrealistic Progress Reporting
Project managers often report incorrect progress percentages.
Example:
Task duration: 100 days
Elapsed time: 28 days
Reported progress: 60%
Actual expected progress should be approximately 28%.
Calculation:
Expected Progress =
(Today − Start Date) / Duration
If difference between expected and reported progress exceeds threshold:
Example threshold:
20–30%
Then flag activity.
Example output:
Task: Cable Installation
Expected progress: 28%
Reported progress: 60%
Status: Progress anomaly.
5.3 Detection Module C
Dependency Chain Analysis
Construction schedules rely heavily on dependencies.
Example:
Cable trays → electrical installation → lighting → controls.
Nova must detect long chains of dependencies and analyze risk propagation.
If a task early in the chain is delayed, downstream tasks will also be affected.
Algorithm steps:
1. Build dependency graph
2. Identify chains of dependencies
3. Detect chains longer than threshold
4. Evaluate delay impact
Output example:
Critical dependency chain detected:
Electrical installation → lighting installation → controls commissioning.
Risk level: High.
5.4 Detection Module D
Decision Activities
Many activities have duration of zero days.
These are typically coordination or decision tasks.
Examples:
Approval of equipment
Component selection
Client decisions
These activities are often hidden schedule risks.
Logic:
If duration = 0 days
AND activity name contains keywords like:
approval
selection
decision
placement
coordination
Then classify as decision activity.
Output example:
Client decision required: Placement of fire safety components.
Risk level: medium.
5.5 Detection Module E
Artificial Scheduling Clusters
In many schedules multiple tasks start on exactly the same date.
Example found in the schedule:
9 tasks starting on the same day.
This often indicates placeholder planning rather than realistic scheduling.
Logic:
If more than X tasks share the same start date within the same discipline:
Flag cluster.
Example output:
Scheduling cluster detected.
8 tasks scheduled to start on Feb 9.
Possible planning placeholder.
5.6 Detection Module F
Long Duration Activities
Tasks exceeding a certain duration often carry higher risk.
Example thresholds:
90 days
120 days
If duration exceeds threshold:
Flag for risk.
Example output:
Task: Project Engineering Completion
Duration: 125 days
Risk level: Elevated.
